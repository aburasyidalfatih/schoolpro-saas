Susun dokumen tersebut dengan struktur sebagai berikut:

1. Ringkasan Eksekutif (Executive Summary):

Status kesehatan kode secara umum (Skor 1-10).

Daftar temuan paling kritis yang harus segera diperbaiki.

2. Detail Temuan Teknikal (Technical Findings):

Gunakan tabel untuk menyajikan temuan. Kolom tabel: Aspek (Fundamental/Security/Performance), Temuan, Tingkat Risiko (Low/Medium/High), dan Status.

3. Analisis Mendalam & Rekomendasi:

Kelompokkan per kategori (Architecture, Security, QA).

Untuk setiap temuan penting, sertakan Blok Kode Rekomendasi (Sebelum vs Sesudah) agar mudah dieksekusi.

4. Checklist Pasca-Audit (Action Plan):

Langkah-langkah taktis yang harus dilakukan di branch feature/audit-refactor sebelum di-merge ke main dan di-deploy ulang via GitHub Actions.

5. Kesimpulan Penutup:

Pernyataan apakah kode ini sudah layak menyandang status 'Production-Ready' atau masih memerlukan perbaikan besar.

Tampilkan dokumen ini dalam format Markdown yang rapi agar saya bisa menyalinnya ke dalam file README_AUDIT.md"
